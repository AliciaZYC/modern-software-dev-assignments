# LLM Power Prompting

## 1. 提示词背景 (Prompting Background)
* **编程的演进：** 提示词是获取 LLM 输出的"通用语言"，也是对其进行有效编程的手段。它被视为编程语言进阶的下一个阶段。
* **自然语言化：** 类比搜索引擎从布尔代数到自然语言的转变，提示词也正变得越来越自然化。
* **艺术与科学的结合：** 虽然 LLM 具有黑盒性质，但已有一系列成熟的技术能显著提升其表现。

## 2. 核心提示词技术 (Core Techniques)

| 技术名称 | 描述 | 适用场景/特点 |
| :--- | :--- | :--- |
| **零样本提示 (Zero-shot)** | 直接向 LLM 下达指令，不提供任何示例。模型完全依赖预训练知识完成任务。 | 简单、通用任务；快速原型验证。 |
| **K样本提示 (K-shot)** | 在提示词中提供 k 个输入输出示例，让模型通过上下文学习归纳规律，无需微调。 | 特定领域 API、自定义代码风格、输出格式严格的任务。 |
| **思维链 (Chain-of-Thought, CoT)** | 通过展示逐步推理过程引导模型"思考后再回答"。零样本版本使用"Let's think step by step"，多样本版本提供完整推理示例链。 | 数学计算、逻辑推理、代码调试等需要中间步骤的复杂任务。 |
| **自一致性 (Self-consistency)** | 对同一问题进行多次独立采样，取多数结果作为最终答案，类似集成学习的思想。 | 减少幻觉、提升答案稳定性；适用于有明确正确答案的推理题。 |
| **工具使用 (Tool Use)** | 为 LLM 配备外部工具（如搜索引擎、代码执行器、数据库、API），让其在推理过程中主动调用。 | 需要实时信息、精确计算或与外部系统交互的 Agent 场景。 |
| **检索增强生成 (RAG)** | 在生成前先从外部知识库检索相关文档片段，将其注入上下文，使模型基于最新、私有或专业知识作答。 | 企业知识库问答、减少幻觉、提供可溯源引用。 |
| **反思 (Reflexion)** | 模型生成初始输出后，对其进行自我批判，识别错误或不足，并迭代修正。可结合外部反馈信号（如代码执行结果）。 | 代码生成与调试、长文档写作、多步骤 Agent 行为的核心机制。 |

## 3. 提示词术语 (Terminology)
* **系统提示词 (System Prompt)：** 设定角色、规则和风格（通常用户不可见）。
* **用户提示词 (User Prompt)：** 人类的具体请求。
* **助手 (Assistant)：** LLM 生成的响应。

## 4. 最佳实践 (Best Practices)
* **清晰性：** 提示词应易于理解，无歧义。
* **角色设定 (Role Prompting)：** 积极使用特定角色（如高级开发者）来增强系统提示。
* **结构化格式：** 使用 XML 风格标签（如 `<log>`, `<error>`）组织输入。
* **明确与拆解：** 明确技术栈约束，并将任务分解。
