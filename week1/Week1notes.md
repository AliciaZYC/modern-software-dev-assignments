## 1. 提示词背景 (Prompting Background)
* **编程的演进：** 提示词是获取 LLM 输出的“通用语言”，也是对其进行有效编程的手段。它被视为编程语言进阶的下一个阶段。 [cite: 2]
* **自然语言化：** 类比搜索引擎从布尔代数到自然语言的转变，提示词也正变得越来越自然化。 [cite: 2]
* **艺术与科学的结合：** 虽然 LLM 具有黑盒性质，但已有一系列成熟的技术能显著提升其表现。 [cite: 2]

## 2. 核心提示词技术 (Core Techniques)

| 技术名称 | 描述 | 适用场景/特点 |
| :--- | :--- | :--- |
| **零样本提示 (Zero-shot)** | 直接向 LLM 下达指令，不提供任何示例。 [cite: 2] | 简单任务。 |
| **K样本提示 (K-shot)** | 提供示例（如 1、3、5 个），也称为“上下文学习”。 [cite: 2] | 特定领域 API、特定代码风格。 |
| **思维链 (Chain-of-Thought, CoT)** | 展示推理步骤。包括零样本“逐步思考”和多样本推理示例。 [cite: 2] | 编程、数学等逻辑任务。 |
| **自一致性 (Self-consistency)** | 多次采样并取多数结果。 [cite: 2] | 减少幻觉。 |
| **工具使用 (Tool Use)** | 允许 LLM 与外部系统交互。 [cite: 2] | 增强自主性。 |
| **检索增强生成 (RAG)** | 注入上下文数据。 [cite: 2] | 提供引用，减少幻觉。 |
| **反思 (Reflexion)** | LLM 对输出进行自我批判和修正。 [cite: 2] | 代理行为的核心。 |

## 3. 提示词术语 (Terminology)
* **系统提示词 (System Prompt)：** 设定角色、规则和风格（通常用户不可见）。 [cite: 2]
* **用户提示词 (User Prompt)：** 人类的具体请求。 [cite: 2]
* **助手 (Assistant)：** LLM 生成的响应。 [cite: 2]

## 4. 最佳实践 (Best Practices)
* **清晰性：** 提示词应易于理解，无歧义。 [cite: 2]
* **角色设定 (Role Prompting)：** 积极使用特定角色（如高级开发者）来增强系统提示。 [cite: 2]
* **结构化格式：** 使用 XML 风格标签（如 `<log>`, `<error>`）组织输入。 [cite: 2]
* **明确与拆解：** 明确技术栈约束，并将任务分解。 [cite: 2]
